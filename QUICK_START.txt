   ADMIN_PASSWORD=your_secret_password

🚀 HOW TO USE
=============

Admin User Management (/su):
  1. Go to: https://yoursite.com/su
  2. Enter admin password
  3. Click "Users" tab
  4. Options:
     - Create User: Enter email, select role (default: user), click Create
     - Edit User: Click edit icon, modify fields, save
     - Delete User: Click trash icon, confirm

User Signup (with referral code):
  1. User visits signup page with referral code
  2. System checks code role:
     - Admin code → new user gets admin role
     - User code → new user gets user role
  3. User automatically appears in admin panel

📊 DEFAULT BEHAVIOR
====================

Action                          | Role    | Can Change?
Sign up without referral        | user    | No
Sign up with admin code         | admin   | No (code determines)
Sign up with user code          | user    | No (code determines)
Admin adds user via /su         | user    | Yes (dropdown)

🔐 AUTHENTICATION
=================

Admin Panel (/su):
  - Uses: Admin password (Bearer token)
  - Required: ADMIN_PASSWORD environment variable

User Signup:
  - Uses: Supabase Auth
  - Role determined by: Referral code (if used) or defaults to 'user'

⚠️  IMPORTANT REMINDERS
========================

2. Set ADMIN_PASSWORD in environment

   - Use a strong password (30+ characters recommended)

   - Go to /su → Referrals tab
   - Create code with role='admin'
   - Users signing up with this code become admins

   - You can change it in dropdown during creation
   - Or edit user after creation

🆘 TROUBLESHOOTING
===================

Problem: Users not appearing after signup
Solution: 
  - Run SUPABASE_AUTH_TRIGGER.sql
  - Check Supabase SQL for trigger errors

Problem: /su page shows "Invalid password"
Solution:
  - Verify ADMIN_PASSWORD is set
  - Check for leading/trailing whitespace
  - Ensure password is correct

Problem: Can't create users in /su
Solution:
  - Check SUPABASE_SERVICE_ROLE_KEY is set
  - Verify Supabase connection works (check logs)

Problem: User roles not set correctly
Solution:
  - Verify referral code role is correct
  - Check auth metadata in Supabase
  - Verify trigger is running
